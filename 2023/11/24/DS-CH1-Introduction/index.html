迴圈／遞迴 (recursive)

常見考題

  1. 寫出遞迴
  2. 遞迴轉非遞迴
  3. 追蹤出結果為何
  4. 呼叫次數、時間複雜度

Recursive種類

遞迴(Recursion)的概念是將一個大的問題，分割成許多小問題去解決。而從程式設計角度來看，函式不單只能被其他函式呼叫，也能被它自己呼叫，也就是在一個函式當中呼叫它自己，即為Recursive Function。

一函式執行過程時反覆做自我呼叫(self-calling)

  1. direct: 呼叫自己。
  1. indirect: 先呼叫其他，其他在呼叫自己。不易debug！
  1. tail: 在函式的最後一行呼叫自己。

建議改成非遞迴（while/for loop）以提高系統效能。

Recursive （空間少、執行慢） Non-Recursive （空間大、執行快）
複雜的演算法時程式簡潔，如DFS、Quick Sort。 code冗長
易理解 不易理解
程式佔記憶體空間少 程式佔記憶體空間大
暫存變數少 暫存變數多
需stack支援 不需stack支援
執行時間長，因為需要不斷傳遞參數， 並且儲存與讀取 return 位置和保存暫存器狀態。(function calling) 執行時間短

Function Calling

  1. 保存funA執行狀況（參數值、區域變數、返回位址）。
  2. 控制權移轉：花時間做data和指令的控制權移轉。
  3. 若stack為空，則程式結束，否則pop1/2，go to 2。

越晚被呼叫越早結束（stack後進先出）

資料型態（Data Type）

由一組data object和作用在這些object上的操作所組成。
例如：
int: data type
⇒{object: 0−9operation:+−∗/\Rightarrow \begin{cases} object:\ 0-9 \\ operation: +-*/ \end{cases}⇒{object: 0−9operation:+−∗/​

抽象資料型態(Abstact Data Type, ADT):

只提供規範，而非實作（只定義不實作）
特色：

  1. ADT也是一種資料型態
  2. 提供操作介面和製作細節的獨立性
  3. 利用ADT可協助定義出所需的class
  4. 使系統具高維護性和彈性

stack的ADT:

  1. 滿足後進先出
  2. 有top指標
  3. 操作包含pop/push…
