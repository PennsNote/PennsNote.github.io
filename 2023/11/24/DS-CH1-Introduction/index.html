Recursive種類

遞迴(Recursion)的概念是將一個大的問題，分割成許多小問題去解決。而從程式設計角度來看，函式不單只能被其他函式呼叫，也能被它自己呼叫，也就是在一個函式當中呼叫它自己，即為Recursive Function。

一函式執行過程時反覆做自我呼叫(self-calling)

  1. direct: 呼叫自己。
  2. indirect: 先呼叫其他，其他在呼叫自己。不易debug！
  3. tail: 在函式的最後一行呼叫自己。

建議改成非遞迴（while/for loop）以提高系統效能。

Recursive （空間少、執行慢） Non-Recursive （空間大、執行快）
複雜的演算法時程式簡潔，如DFS、Quick Sort。 code冗長
易理解 不易理解

程式佔記憶體空間少 | 程式佔記憶體空間大
暫存變數少 | 暫存變數多
需stack支援 | 不需stack支援
執行時間長，因為需要不斷傳遞參數，並且儲存與讀取 return 位置和保存暫存器狀態。(function calling)| 執行時間短

Function Calling

  1. 保存funA執行狀況（參數值、區域變數、返回位址）。
  2. 控制權移轉：花時間做data和指令的控制權移轉。
  3. 若stack為空，則程式結束，否則pop1/2，go to 2。

:bulb: 越晚被呼叫越早結束（stack後進先出）

  3. 常用 T(n) 代表時間函數。

漸進式相加

例題1. $\Omega(n)+\theta(n^2)$
:::spoiler
<證明> $\Omega(n)+\theta(n^2) = [n,∞]+[n^2,n^2] \ 取大的 \
= [n^2,∞] \ 只知道下限 \Rightarrow \Omega(n^2)$
:::

例題2. $\Omega(n)+o(n^2)$
:::spoiler
<證明> $\Omega(n)+o(n^2) = [n,∞]+[1,n^2] \ 取大的 \
= [n,∞] \ 只知道下限 \Rightarrow \Omega(n^2)$
:::

例題3. $\theta(n)+o(n^2)$
:::spoiler
<證明> $\theta(n)+o(n^2) = [n,n]+[1,n^2] \ 取大的 \
= [n,n^2] \Rightarrow o(n^2) \ or \ \Omega(n)$
:::

常見考型

:bulb: 替代展開法

例題1. 河內塔

$T(n)=2T(n-1)+1, \ T(1)=1, \ T(0)=0$
:::spoiler
$2T(n-1)+1 \ =2[2T(n-2)] \ = \ 4T(n-2)+3 \ =4[2T(n-3)+1]+3 \ = \ 8T(n-3)+7 \ …=2^nT(n-n)+(2^n-1) \ = \ 0+(2^n-1) \Rightarrow o(2^n)$
:::

:bulb: 支配理論

Divide-and-Conquer之時間函數
(1) divide: 將一大問題分成多個小問題
(2) conquer: 針對每個小問題求解
(3) combine: 合併後求得最後解

$T(n)=aT(\frac{n}{b})+f(n)$

  1. $n^{\log_b{a}}>f(n) \Rightarrow \ T(n)=\theta(n^{\log_b{a}})$
  2. $n^{\log_b{a}}=f(n) \Rightarrow \ T(n)=\theta(n^{\log_b{a}}*logn)$
  3. $n^{\log_b{a}}<f(n) 且 \ af(\frac{n}{b}) \le \ cf(n) \ 其中c < １ \ \Rightarrow \ T(n)=\theta(f(n))$

:bulb: 延伸支配理論(Extended Master Method):

當$f(n)$裡面有$\log{n}$又有$n^{\log_b{a}}$
若 $T(n)=aT(\frac{n}{b})+n^{log_b{a}}log^k{n} \ \Rightarrow T(n)=\theta(n^{log_b{a}}log^{k+1}{n})$

:bulb: Recursive Tree

若$level cost=n, \ T(n)=o(f(n)\log{n})$
若$level cost<n, \ T(n)=o(f(n))$

抽象資料型態(Abstact Data Type, ADT):

只提供規範，而非實作（只定義不實作）
特色：

  1. ADT也是一種資料型態
  2. 提供操作介面和製作細節的獨立性
  3. 利用ADT可協助定義出所需的class
  4. 使系統具高維護性和彈性 stack的ADT:
  5. 滿足後進先出
  6. 有top指標
  7. 操作包含pop/push…
